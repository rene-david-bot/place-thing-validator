Real PoC setup

Use your own OpenAI key for live multimodal testing. Nothing is committed to GitHub, and key persistence is optional.

OpenAI settings

Recommended starting model: gpt-5.4. Change it if your account exposes a better multimodal model.

If remember is off, the key stays only in memory until reload. If remember is on, it is stored only in this browser. It is never committed to the repo. This is still a client-side key, so use a test key you control. Blind mode keeps manual object/place labels and notes out of AI prompts by default, so the model leans more on images and blind place-memory cues.

What this PoC validates

Stage 1, place memory

  • Capture overview, alternate, detail, and hard shots
  • Generate structured place cues with AI
  • Build a reusable place memory index

Stage 2, object in context

  • Capture the object with surrounding background context
  • Infer likely place using mapped place memory
  • Track confidence, ambiguity, and errors

Search

  • Search the captured dataset with natural-language descriptions
  • Score ranked results against expected truth
  • Review whether the concept is good enough for MVP

Local-first PoC

  • Images and app state stay in the browser by default
  • No backend or secret committed to GitHub Pages
  • Mobile camera capture works directly in the browser

Add mapped place

Capture guidance

  • Keep one wide overview for scene identity.
  • Move left or right for the second angle, do not just zoom.
  • Show shelf edges, handles, workbench context, labels, or clutter.
  • Use the hard-mode shot when the place is often messy or dim.
  • Do not submit four nearly identical photos.

Quick-save guidance

  • Start with one context photo only. That is the default happy path.
  • AI should prefill object name, place, and attributes, then the user can edit before saving.
  • If the app is unsure, ask for more context, a second photo, or place capture instead of forcing a bad save.
  • Later search should work from natural descriptions, not exact labels.
